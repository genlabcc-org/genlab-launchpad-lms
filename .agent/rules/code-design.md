## code-design

Rules:
- While writing any code in this project, you MUST strictly adhere to the following software design principles:
  1. **SOLID** (High Priority): Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, and Dependency Inversion. Ensure classes, modules, and services have a single responsibility and are decoupled.
  2. **KISS** (Keep It Simple, Stupid): Avoid over-engineering, unnecessary abstractions, and redundant complexity. Simple code is readable and maintainable.
  3. **DRY** (Don't Repeat Yourself): Avoid duplication by extracting common logic, styling, and data processing.
