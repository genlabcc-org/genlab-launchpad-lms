## Why

We need to verify the correctness of our frontend API layer without manually testing endpoints or running the live backend server. By introducing Vitest as our test runner and MSW to intercept HTTP requests at the network level, we can write robust, isolated integration and unit tests for our frontend services. 

Furthermore, to prevent the mock endpoints from drifting out of sync with the Java Spring Boot backend, we must automate the generation of TypeScript types and MSW handlers directly from an OpenAPI/Swagger definition of the backend.

## What Changes

We will introduce a fully automated testing and type-generation pipeline:
1. Configure `springdoc-openapi-starter-webmvc-ui` in the backend `pom.xml` and include a plugin to write the OpenAPI JSON file to disk during compilation.
2. Install `vitest`, `msw`, `msw-auto-mock`, and `openapi-typescript` in the frontend `devDependencies`.
3. Set up Vitest mock server initialization in `src/api/setupTests.ts` and add automation scripts to `package.json` to generate TS interfaces and MSW mock handlers.
4. Keep the design simple (KISS), reusable (DRY), and decoupled (SOLID), ensuring the frontend continues to communicate exclusively with the backend REST endpoints.

## Capabilities

- `frontend-testing-automation`: Automatic generation of API mock handlers and types from OpenAPI spec, combined with Vitest + MSW testing execution.

## Impact

- **Backend**: Adds OpenAPI generation dependencies and plugin. Outputs `target/openapi.json`.
- **Frontend**: Adds testing scripts (`test`, `openapi:sync`), configuration files (`vitest.config.ts`, `setupTests.ts`), and autogenerated types and mock handler directories.
